# 蜘蛛纸牌 — 手动 UI 测试清单

自动化测试覆盖了游戏逻辑和 UI 计算，但以下场景需要在真机（Huawei Mate 40）
或 Kivy 桌面预览窗口中手动验证。

---

## 1. 启动与菜单

- [ ] App 启动无崩溃，菜单界面正常显示
- [ ] 三个难度按钮（初级/中级/高级）文字清晰，无乱码
- [ ] "历史记录"按钮可点击，跳转到统计界面
- [ ] "继续游戏"按钮在有存档时出现，无存档时隐藏
- [ ] 所有中文字符正确显示（非方块 □）

## 2. 牌桌布局

- [ ] 10 列卡牌全部可见，不溢出屏幕
- [ ] 左 4 列各 6 张牌，右 6 列各 5 张牌
- [ ] 每列最底部的牌正面朝上，其余面朝下
- [ ] 面朝下的牌显示蓝色背面 + 交叉纹样
- [ ] 面朝上的牌显示白色正面 + 等级 + 花色
- [ ] 花色颜色正确：♥♦ 红色，♠♣ 黑色
- [ ] J 和 I 可以区分（字体大小足够）

## 3. 拖放操作

- [ ] 点击面朝上的牌可以开始拖拽
- [ ] 拖拽中的牌在最上层（不被其他牌遮挡）
- [ ] 拖拽序列保持正确间距
- [ ] 松手时自动吸附到最近的合法目标列
- [ ] 松手位置偏离目标中心但在容差内仍能放下
- [ ] 非法移动时牌自动归位
- [ ] 点击面朝下的牌无反应

## 4. 游戏规则

- [ ] 同花色递减序列可以一起移动
- [ ] 不同花色的序列不能一起移动
- [ ] 任何牌都可以放到空列（不限 K）
- [ ] 发牌前必须填满所有空列（有提示文字）
- [ ] K→A 同花色完整序列自动收集到完成区

## 5. 动画

- [ ] 发牌动画：10 张牌从右下角发牌堆依次飞出
- [ ] 完成动画：13 张牌依次飞向左下角完成区
- [ ] 动画期间触摸被屏蔽（不能拖牌/发牌）
- [ ] 动画结束后操作恢复正常

## 6. 状态栏与按钮

- [ ] 分数、步数、时间、难度标签持续更新
- [ ] 时间格式 MM:SS 正确（有前导零）
- [ ] "撤销"按钮恢复上一步（牌、分数、步数）
- [ ] "新游戏"弹出确认对话框
- [ ] "菜单"按钮回到主菜单

## 7. 胜利界面

- [ ] 8 组完成后弹出胜利对话框
- [ ] 显示最终得分、步数、用时
- [ ] "再来一局"开始新游戏
- [ ] "菜单"回到主菜单

## 8. 统计界面

- [ ] 显示全部/按难度的统计数据
- [ ] 数字格式正确（胜率有 %，时间有 m:ss）
- [ ] 可滚动（记录多时）
- [ ] "返回"按钮回到菜单

## 9. 数据持久化

- [ ] 关闭 app 重新打开后统计数据保留
- [ ] 卸载重装后统计数据从 /sdcard/ 备份恢复
- [ ] 版本升级后数据正确合并（无重复）

## 10. 屏幕适配

- [ ] 竖屏布局正确
- [ ] 横屏（如果允许旋转）布局不崩溃
- [ ] 完成区 8 个位置不溢出
- [ ] 发牌堆显示剩余组数

## 11. 难度随机性

- [ ] 初级：每局只出现 1 种花色，花色每局可能不同
- [ ] 中级：每局出现 2 种花色，花色组合可能变化
- [ ] 高级：每局固定 4 种花色

---

**测试方法建议**:
1. 桌面预览：`python main.py`（窗口 420×920 模拟 Mate 40）
2. 真机测试：安装 APK 后按以上清单逐项检查
3. 每次改动 UI 代码后至少过一遍第 2-5 节
