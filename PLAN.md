# 蜘蛛纸牌 — 详细开发计划

## 1. 项目目标

开发一款可以在华为 Mate 40 手机上运行的经典蜘蛛纸牌游戏 APK。

- 免费、无广告、无版权问题
- 支持三种难度（初级/中级/高级）
- 经典简洁视觉风格
- 全 Python 技术栈

---

## 2. 技术方案

### 2.1 为什么选 Kivy + Buildozer

| 选项 | 优点 | 缺点 |
|------|------|------|
| **Kivy + Buildozer（选定）** | 全 Python、原生触屏、APK 打包成熟 | 包体积较大（~30MB） |
| Pygame + Buildozer | 游戏开发直觉好 | 触屏支持不如 Kivy |
| Flask + WebView | 开发快 | 用户体验差、需要前端代码 |

核心依赖（全部 MIT 许可，无版权风险）：
- Python 3.10+
- Kivy 2.3.1（GUI 框架）
- Buildozer 1.5.0（APK 打包）
- Cython（Buildozer 编译需要）

### 2.2 华为 Mate 40 适配

| 参数 | 值 |
|------|------|
| 屏幕 | 6.5 英寸，2376×1080 |
| 系统 | HarmonyOS / EMUI（基于 Android） |
| 架构 | ARM64 (arm64-v8a) |
| Android 版本 | 10+ |

适配策略：
- 使用 Kivy 的 `dp` 单位做 DPI 无关设计
- 桌面调试窗口按 9:19.8 竖屏比例预览
- APK 设置 `android.archs = arm64-v8a`
- `android.minapi = 24`（Android 7.0+）

### 2.3 模拟方案

由于没有实体手机：
1. **桌面预览**：Kivy 应用以 420×920 窗口运行，模拟 Mate 40 竖屏比例
2. **鼠标模拟触屏**：Kivy 天生支持鼠标→触屏映射
3. **Android 模拟器**（可选）：用 Android Studio AVD 配置 Mate 40 参数

## 3. 游戏规则（完整定义）

### 3.1 基本设置

- 使用 **104 张牌**（根据难度决定花色组合）
- **初级**：8 套黑桃（♠），共 104 张
- **中级**：4 套黑桃（♠）+ 4 套红心（♥），共 104 张
- **高级**：2 套四种花色（♠♥♦♣），共 104 张

### 3.2 发牌

- 10 列牌堆：左 4 列各 6 张，右 6 列各 5 张 = 54 张
- 每列只有最底下一张正面朝上
- 剩余 50 张放在右下角待发牌堆（5 组，每组 10 张）

### 3.3 移牌规则

1. 可将牌列最底部的单张牌移到空列
2. 可将牌移到点数比它大 1 的牌上面（不限花色）
3. **同花色、递减排列的连续序列**可作为整体移动
4. 同花色从 K 到 A 的完整 13 张序列自动移除（算完成一组）

### 3.4 发新牌

- 点击右下角待发牌堆
- 每次发 10 张（每列 1 张）
- **前提**：所有 10 列必须非空

### 3.5 胜利条件

- 8 组（K→A）全部收完

### 3.6 计分

- 初始 500 分
- 每次移牌 -1 分
- 每完成一组 +100 分

## 4. 项目架构

```
Spider_Solitaire_toy/
├── main.py                    # 应用入口（Kivy App + ScreenManager）
├── buildozer.spec             # Android APK 打包配置
├── spider_solitaire/
│   ├── __init__.py
│   ├── game/                  # 游戏逻辑层（纯 Python，不依赖 UI）
│   │   ├── __init__.py
│   │   ├── card.py            # Card 类：suit, rank, face_up
│   │   ├── deck.py            # 牌组创建和洗牌
│   │   ├── game_state.py      # GameState：核心状态机
│   │   └── rules.py           # 规则验证函数
│   └── ui/                    # Kivy UI 层
│       ├── __init__.py
│       ├── theme.py           # 颜色、尺寸常量
│       ├── card_widget.py     # 单张卡牌 Widget（Canvas 绘制）
│       ├── board_widget.py    # 棋盘 Widget（10 列 + 触摸拖拽）
│       ├── game_screen.py     # 游戏主界面（状态栏 + 按钮）
│       └── menu_screen.py     # 菜单界面（难度选择）
├── tests/
│   ├── __init__.py
│   └── test_game.py           # 39 个单元测试
├── LICENSE                    # MIT 许可证
└── README.md                  # 项目说明
```

### 4.1 分层设计

```
┌─────────────┐
│   main.py   │  应用入口、屏幕切换、存档管理
├─────────────┤
│  UI 层      │  Kivy Widget：卡牌绘制、棋盘布局、触摸交互
├─────────────┤
│  逻辑层     │  纯 Python：规则验证、状态管理、撤销、序列化
└─────────────┘
```

UI 层只负责显示和交互，所有规则判断都在逻辑层完成。这样便于单元测试和后期维护。

## 7. 关键设计决策

1. **为什么不用外部图片素材？** → 避免版权风险，全部代码绘制
2. **为什么选竖屏为主？** → 手机自然握持方式，但同时支持横屏
3. **为什么用 dp 单位？** → 自动适配不同 DPI 屏幕
4. **为什么逻辑层和 UI 层分离？** → 便于单元测试、后期维护
5. **为什么用 JSON 存档？** → 简单、跨平台、易调试
